[1mNAME[0m[m
       journalctl - Query the systemd journal[m
[m
[1mSYNOPSIS[0m[m
       [1mjournalctl[0m [OPTIONS...] [MATCHES...][m
[m
[1mDESCRIPTION[0m[m
       [1mjournalctl[0m may be used to query the contents of the [1msystemd[0m(1) journal as written by [1msystemd-[0m[m
       [1mjournald.service[0m(8).[m
[m
       If called without parameters, it will show the full contents of the journal, starting with the oldest entry[m
       collected.[m
[m
       If one or more match arguments are passed, the output is filtered accordingly. A match is in the format[m
       "FIELD=VALUE", e.g.  "_SYSTEMD_UNIT=httpd.service", referring to the components of a structured journal entry.[m
       See [1msystemd.journal-fields[0m(7) for a list of well-known fields. If multiple matches are specified matching[m
       different fields, the log entries are filtered by both, i.e. the resulting output will show only entries[m
       matching all the specified matches of this kind. If two matches apply to the same field, then they are[m
       automatically matched as alternatives, i.e. the resulting output will show entries matching any of the[m
       specified matches for the same field. Finally, the character "+" may appears as a separate word between other[m
       terms on the command line. This causes all matches before and after to be combined in a disjunction (i.e.[m
       logical OR).[m
[m
       As shortcuts for a few types of field/value matches, file paths may be specified. If a file path refers to an[m
       executable file, this is equivalent to an "_EXE=" match for the canonicalized binary path. Similarly, if a[m
       path refers to a device node then match is added for the kernel name of the device ("_KERNEL_DEVICE="). Also,[m
       matches for the kernel names of all the parent devices are added automatically. Device node paths are not[m
       stable across reboots, therefore match for the current boot id ("_BOOT_ID=") is always added as well. Note[m
       that only the log entries for the existing device nodes maybe queried by providing path to the device node.[m
[m
       Additional constraints may be added using options [1m--boot[0m, [1m--unit=[0m, etc, to further limit what entries will be[m
       shown (logical AND).[m
